# XTATION User Journey V1

## Purpose

This document defines how a real user should experience Xtation.

It answers:

- who the first users are
- what they see first
- how onboarding should work
- how the daily loop should work
- how the product should reveal complexity gradually

This is the experience layer above the architecture docs.

## Core Principle

Xtation should feel deep, but it should not feel hard to enter.

The right experience model is:

- immediate value first
- depth second
- customization third

Users should not meet the entire system on day one.

## First User Profile

The first ideal Xtation user is likely one of these:

- focused solo builder
- creator/operator
- founder
- personal systems enthusiast
- ambitious student or self-directed learner

These users want:

- structure
- momentum
- memory
- better execution
- guidance without losing control

## First Impression Rule

Within the first minute, the user should understand:

- this is their personal station
- it helps them do real work
- it remembers their context
- it can grow into something deeper later

The first impression should not be:

- too many tabs
- too many empty sections
- unexplained jargon
- a giant assistant chat

## Primary Journey

The main Xtation user loop should be:

```txt
Sign up
-> quick setup
-> first quest
-> first session
-> first reward
-> return tomorrow
```

If this loop works, Xtation has a base product.

## 1. Landing and Signup

The public site should explain Xtation fast.

Best message:

- plan less vaguely
- execute more clearly
- track growth like a living system

The signup flow should be simple.

Recommended first options:

- email sign up
- email sign in
- magic link or password
- social auth later only if it clearly helps

## 2. Onboarding

Onboarding should be short and purposeful.

Recommended onboarding flow:

### Step 1: Intent

Ask:

- what are you trying to improve right now

Examples:

- focus
- learning
- building
- health
- organization

### Step 2: Self Tree Seed

Ask the user to choose or confirm a few growth areas inside the 6 fixed branches.

Do not make them build the full self tree.
Just seed it.

### Step 3: First Quest

Help them create one clear first quest.

Examples:

- finish one study session
- write one draft
- do one workout
- build one feature

### Step 4: First Session

Guide them directly into Play.

This is the first real proof of value.

### Step 5: First Reward

When they finish, show:

- XP
- progress
- next suggestion
- optional Dusk reflection

## 3. Default Post-Onboarding Home

The default home should not be a generic dashboard.

Recommended logic:

- if there is an active quest or session, open `Play`
- otherwise show a calm `Play-ready` home with:
  - current priority
  - next action
  - quick start
  - Dusk brief

This keeps the product action-oriented.

## 4. Daily Use Loop

The normal returning-user loop should be:

```txt
Open Xtation
-> see active priority
-> enter Play
-> run session
-> complete or pause
-> review outcome
-> optionally refine system in Lab
```

That is the spine.

## 5. Progressive Disclosure

Xtation should reveal complexity gradually.

### Day 1

Focus on:

- Play
- basic Profile stats
- very light Dusk presence

### Day 2-7

Introduce:

- Lab notes/templates
- self-tree expansion
- inventory/resource linking

### Later

Introduce:

- deeper automations
- assistant projects
- multiplayer/people features
- skins/store

This prevents overwhelm.

## 6. Dusk in the Journey

Dusk should help the user move, not stop them.

Recommended first Dusk roles:

- explain what to do next
- help create first quest
- help summarize first session
- suggest one useful Lab or Profile action

Dusk should not dominate onboarding with long conversation.

## 7. Lab in the Journey

Lab should not appear as a huge complex area on first use.

First useful Lab moments:

- save a note
- use a template
- create one simple automation later

Only after the user has already felt value in Play.

## 8. Profile in the Journey

Profile should feel rewarding and clear.

Users should quickly understand:

- level
- streak
- recent progress
- self-tree growth

Profile should feel like a mirror of action, not a static social page.

## 9. Multiplayer in the Journey

Multiplayer should not be part of first-run complexity unless the product is explicitly launched with a people-first positioning.

Recommended first use later:

- add one person
- create one focus room
- share one quest or accountability session

This keeps the system grounded in use.

## 10. Store in the Journey

Store should not interrupt first-value creation.

Best timing:

- after the user understands the core
- after they care about identity/customization
- after templates/skins/modules actually mean something

Store should feel like expansion, not pressure.

## 11. Friction Rules

Xtation should aggressively reduce these forms of friction:

- too many empty surfaces
- forcing setup before action
- making the user choose too many categories up front
- asking for details that can be inferred later
- using game language without practical meaning

## 12. Delight Rules

Xtation should create delight through:

- clarity
- smooth execution
- meaningful feedback
- thoughtful Dusk guidance
- calm visual identity
- moments of atmosphere in the right places

Not through:

- noise
- over-animation
- random gamification
- too many badges or fake urgency

## 13. Recommended First-Run UI Structure

The first-run app experience should be close to:

```txt
Welcome
-> Quick setup
-> Seed self tree
-> Create first quest
-> Start first session
-> Finish
-> See profile and next suggestion
```

This is enough.

## 14. Retention Hooks

The strongest reasons to come back should be:

- unfinished active quest
- streak or momentum
- self-tree growth
- Dusk brief
- useful note/template/automation memory
- progress you can feel in Profile

These are better than generic notification spam.

## 15. Monetization Fit

The product should monetize after users believe:

- this helps me work better
- this remembers my system
- this grows with me

Then premium can feel natural through:

- connected Dusk
- deeper Lab
- richer skins
- more storage
- premium packs

## 16. Final Lock

Xtation should introduce itself like this:

- help the user take action
- show that action becomes progress
- show that progress becomes identity
- then open the deeper systems

That is the simplest and strongest experience path.
